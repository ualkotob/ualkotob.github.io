Although JavaScript isnt inherently a functional programming language, we can use some of the practices of function programming to improve our code.

Functions Are Self-Contained

We have a simple function sum here. It adds all the number of the numArray then prints the result. This function is breaking a bunch of function programming rules. The first we'll address is being self-contained. Functions should have no hidden inputs/outputs. Every input should be a parameter, and every output should be returned. Currently our function has no parameters and returns nothing. Let's fix that.

Now we have it take the numArray as a parameter, and return the console.log. Not we know this function takes in a number array and returns a print to console. Before this information was hidden in the function.

Functions Do One Thing

Although our function may seem simple, it still is doing too much. Function should just do one thing, and out function does two. It add the numbers and prints to the console. We can fix that too.

Not we have the function split into sum and printResult. They each just do one thing, and are still self-contained. printResult is kind of a pointless function, but it is a good example of functional programming practices.

Functions Are Recursive

The final concept Im going over is recursion. Although not always necessary, making a function recursive is more in line with functional programming paradigms. Our sum function includes a for loop where we iterate over a list. Were going to replace this with recursion.

So, not instead of using a for loop, we are using an index to track our progress thought the array, and then calling our function in itself to add the rest. Recursion is when a function calls upon itself. For every value in the array, out new function will make another call adding each subsequent return value to the total.

Conclusion

Function programming isnt a take all or nothing thing in JavaScript. In general, it breaks down to all functions should always have the same output for the same input. All of these concepts above help achieve this, and get us closer to this goal. Its our job as the programmer to decide when it's best to employ these.
