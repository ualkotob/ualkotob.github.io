Dependency injection is the practice of putting all of a functions dependency's in its parameters. The makes it clear what data the function is using when we call it, everything it uses needs to be passed in.

Why Use Dependency Injection?

Although designing your functions this way can make them more flexible, the main reason it is used is for testing. When a function has external dependencies, it can be a nightmare to test. When it throws an error, you can't be sure if there is something wrong with the function, or one of its dependencies. When all dependencies are in the parameters, you can use dummy data for testing, so you can reduce issues. This is often done using XML to test functions and find out if it can deal with every situation required of it.

Conclusion

We can use dependency injection to make our functions easier to debug. It can also help with testing different data from external sources.
