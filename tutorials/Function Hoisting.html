Function hoisting is the JavaScript paradigm where function and variable declarations move to the top of the current scope at runtime. Before we can understand this, we need to understand how scope works in JavaScript.

Scope

Functions create scope in JavaScript, so whenever you declare a new function, you are creating a new scope.

In this example we have three scopes. Scope A is the global scope. Scope B is created when we declare the doSomething function. Then we have Scope C which is created when we declare the doSomethingElse function. Scopes can access other scopes as long as their function is within that Scope.

Scope A cannot access Scope B and Scope C. Scope B cannot access Scope C. Scopes can only access items in their scope, and in outer scopes. They cannot access in inner scopes.

Function Hoisting

As you can see, doSomethingElse is called at the beginning of doSomething before it is declared. It still runs, because doSomethingElse had it's declaration moved to the top at runtime. This is function hoisting.

Declarations and Assignments

Only declarations are moved to the top. If a declaration is on the same line as an assignment, the assignments still acts as it would without being moved to the top.

Conclusion

Although it is important to understand how function hoisting works, it is still best practice to make our function declarations at the top. We want to make our code readable, and don’t want to organize it in a way that is unintuitive.
