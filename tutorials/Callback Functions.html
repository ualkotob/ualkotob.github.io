In JavaScript functions are considered objects. This means we can treat them like objects. Callback functions are functions that are passed through other functions.

Why Use Callbacks?

The main reason to use callback functions, it to prevent blocking. JavaScript is inherently a single threaded language, and when a process doesn't let anything else happen until it is finished, it is blocking. A common function that blocks is the alert function. It requires user input before it will let any other processes run. Callback functions wait for the function calling it to complete before executing in the main thread, so we can prevent blocking by using them.

Conclusion

Whenever we pass a function as a parameter of another function, we are using callbacks. These are mainly used to prevent blocking and make our programs asynchronous.
